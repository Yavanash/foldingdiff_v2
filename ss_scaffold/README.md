# ss_scaffold

Secondary-structure-conditioned motif scaffolding on top of FoldingDiff.

Academic project (BT305). Builds a small extension of `foldingdiff/` that lets
you say *"I have this helix, design me a 60-residue protein around it"* and get
back a backbone where the helix is preserved and the surrounding residues are
generated by a denoising diffusion model.

## What's reused vs. what's new

| Component | Source | Status |
|----------|--------|--------|
| 6-angle backbone parameterization (phi, psi, omega, tau, CA:C:1N, C:1N:1CA) | `foldingdiff/datasets.py` | reused as-is |
| BERT-on-continuous-inputs transformer | `foldingdiff/modelling.py` (`BertForDiffusion`) | reused via subclass |
| Diffusion noise schedules | `foldingdiff/beta_schedules.py` | reused |
| Forward q_sample / reverse p_sample | `foldingdiff/sampling.py`, `foldingdiff/datasets.py` | reused (q_sample helper re-derived in `ss_scaffold/sampling.py`) |
| NeRF coordinate reconstruction | `foldingdiff/nerf.py`, `foldingdiff/angles_and_coords.py` | reused |
| Angular loss functions | `foldingdiff/losses.py` | reused via parent's `loss_func` |
| Lightning training loop | `foldingdiff.modelling.BertForDiffusion` | reused via subclass |
| **3-state SS labels (DSSP, H/E/C/PAD)** | `ss_scaffold/ss_labels.py` | **new** |
| **Motif span sampling** | `ss_scaffold/ss_labels.py` | **new** |
| **SS-annotated dataset wrapper** | `ss_scaffold/dataset.py` | **new** |
| **Ramachandran-region prior loss** | `ss_scaffold/losses.py` | **new** |
| **`BertForSSConditionedDiffusion`** (additive SS embedding + is_motif channel) | `ss_scaffold/model.py` | **new** |
| **RePaint-style motif inpainting at sampling time** | `ss_scaffold/sampling.py` | **new** |

## Architecture

```
   angles (B, L, 6)        is_motif (B, L, 1)        ss_labels (B, L)
        \                       /                          |
         concat -----------------                          |
                 |                                         |
                 v                                         v
           inputs_to_hidden_dim                      ss_embedding
           (Linear 7 -> hidden)                      (nn.Embedding)
                 |                                         |
                 +-- + position_embed -- + time_embed -----+
                                 |
                                 v
                            BertEncoder
                                 |
                                 v
                           token_decoder
                                 |
                                 v
                    predicted noise (B, L, 6)
```

At sampling time, motif positions are clamped to a fresh forward-noised version
of the true motif angles at every reverse-diffusion step (RePaint).

## Three design decisions worth knowing

1. **Anchored inpainting, not classifier-free.** The motif is given as actual
   phi/psi/omega/tau angles, not a soft "build a helix here" label. This is
   closer to RFdiffusion motif scaffolding than to per-residue SS conditional
   generation.
2. **Additive SS embedding, not cross-attention.** SS labels are aligned 1:1 to
   residue positions; cross-attention degenerates to a per-token lookup that
   addition does for free. Saves ~70 lines and 30-50% of compute.
3. **Ramachandran-region loss instead of a sequence input.** No amino-acid
   conditioning. Biophysics enters the model as a per-SS-class Gaussian prior
   on (phi, psi). It's the minimum amount of "protein knowledge" that still
   counts as protein knowledge.

## Files

- `__init__.py` -- public API
- `ss_labels.py` -- DSSP wrapper, motif span sampling
- `dataset.py` -- `SSAnnotatedAnglesDataset` wraps `NoisedAnglesDataset` and adds SS + motif fields
- `losses.py` -- `ramachandran_loss` (Gaussian mixture per SS class)
- `model.py` -- `BertForSSConditionedDiffusion` (subclasses `BertForDiffusion`)
- `sampling.py` -- `p_sample_loop_with_motif` (RePaint clamp)
- `train.py` -- training entrypoint
- `sample.py` -- inference entrypoint

## Usage

### Training

```bash
python -m ss_scaffold.train \
    --pdb-dir data/cath_s40 \
    --out runs/ss_scaffold_v1 \
    --epochs 100 \
    --rama-lambda 0.1
```

The dataset wrapper computes DSSP labels for every PDB on first run and caches
them at `<cache-dir>/ss_labels_v1.pkl`.

### Sampling around a known helix

```bash
python -m ss_scaffold.sample \
    --model-dir runs/ss_scaffold_v1 \
    --motif-pdb data/example.pdb \
    --motif-range 10-30 \
    --motif-class H \
    --total-length 80 \
    --n-samples 8 \
    --out generated/
```

The motif's true angles are extracted from `data/example.pdb` residues
[10, 30) and clamped at every reverse-diffusion step. The other 60 residues
are denoised by the model.

## Manual verification plan

No unit tests in this project. Instead, run these checks as you implement /
debug. Each one catches a specific failure mode silently:

1. **Ramachandran loss basins.**
   ```bash
   python -m ss_scaffold.losses
   ```
   Prints the loss surface on a 13x13 (phi, psi) grid for H and E classes.
   Helix minimum should land near phi=-60deg, psi=-45deg. Sheet minimum near
   phi=-120deg, psi=+130deg. If the basins are in the wrong quadrant, the
   degree/radian conversion is wrong.

2. **DSSP sanity.**
   ```bash
   python -m ss_scaffold.ss_labels path/to/example.pdb
   ```
   Prints the 3-state SS string and lists H/E runs of length >=6. Spot-check
   against the structure in PyMOL.

3. **Motif clamp invariant.** Inside `p_sample_loop_with_motif`, the motif
   angles should stay close to ground truth at every step. Add a temporary
   `print(img[:, motif_start:motif_end, :])` at the end of the loop and verify
   they match `motif_angles` to within numerical noise at t=0.

4. **No-motif equivalence.** Run sampling with `motif_mask` all zeros. Output
   should look like vanilla FoldingDiff samples (Ramachandran 2D histogram of
   the result should match what you'd see from the base model on the same
   noise seed).

5. **End-to-end.** Sample around a held-out PDB helix, reconstruct via NeRF,
   run DSSP on the result, check that the motif region is still labelled H by
   DSSP. >=90% preservation is the success criterion.

## Known limitations

- Single-chain PDBs only.
- 3-state SS only (no left-handed alpha basin in the Ramachandran loss).
- One motif per protein at training time (sampling supports any number of
  contiguous regions if you set `motif_mask` accordingly).
- No amino-acid sequence conditioning. Sequences would have to be designed
  separately by an inverse-folding model (e.g. ProteinMPNN).
- DSSP is invoked via biotite; install with `pip install biotite`.
